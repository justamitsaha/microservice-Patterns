Important Commands

Kubernetes Basics

These commands help check pod, service, and deployment status. Use them during troubleshooting or verifying your cluster.

kubectl get pods -A
kubectl get svc -A
kubectl describe pod POD_NAME -n NAMESPACE

Port Forwarding

Use these commands to access internal cluster services like MySQL and Redis from your local machine.

kubectl port-forward svc/mysql 3307:3306 -n microservice
kubectl port-forward svc/redis 6380:6379 -n microservice

Docker Commands

Common Docker commands useful for debugging containers, checking logs, and entering running containers.

docker logs -f container-name
docker exec -it container-name sh
